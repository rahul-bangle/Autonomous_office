# Directory Structure

This document provides a map of the Virtual Office codebase.

## Root Level
- `MASTER_CONTEXT.md`: **[CRITICAL]** The "Source of Truth" for project status, rules, and logs.
- `ceo.py`: **[SUPERVISOR]** Standalone autonomous script for heartbeat/health monitor.
- `package.json`: Node dependencies (Vite, React).
- `.agent/workflows/`: Core AI behavior guidelines and execution SOPs.

## Backend `/backend`
- `main.py`: **[ENTRY POINT]** FastAPI server, WebSocket handlers, and CrewAI orchestration logic.
- `task_log.json`: Historical record of agent activities and token usage.

## Frontend `/src` & Root
- `index.html`: Entry page for the web interface.
- `src/`: React source code (App components, UI logic).

## Skills `/skills`
- **Modules**: Each subdirectory contains a `SKILL.md` defining a specific capability (e.g., search, coding, debugging).
- **Dynamic Loading**: Skills are scanned and loaded by the backend at startup.

## Planning `/.planning`
- `codebase/`: **[CURRENT]** Structured documents generated by `/gsd-map-codebase`.
- `STATE.md`: Tracker for current project milestones and progress.

## Others
- `.git/`: Version control history.
- `node_modules/`: Frontend dependencies.
- `dist/`: Compiled production build (via Vite).
